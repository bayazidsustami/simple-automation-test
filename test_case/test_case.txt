Test Case / Test Scenario Document
Fitur: Login
1. Test Case 1: Valid Credentials
Objective: To verify that a user can successfully login with valid credentials.

Preconditions:

The user is on the login page of the application.
Test Steps:

Enter valid username.
Enter valid password.
Click on the "Login" button.
Expected Result:

The user should be redirected to the home page/dashboard.
Postconditions:

The user is logged in successfully.
2. Test Case 2: Invalid Credentials
Objective: To verify that a user cannot login with invalid credentials.

Preconditions:

The user is on the login page of the application.
Test Steps:

Enter invalid username.
Enter invalid password.
Click on the "Login" button.
Expected Result:

An error message should be displayed indicating that the login failed.
Postconditions:

The user remains on the login page.
Fitur: Add to Cart
1. Test Case 1: Add Item to Cart
Objective: To verify that a user can add items to the cart.

Preconditions:

The user is logged in and on the product page.
Test Steps:

Select a product.
Click on the "Add to Cart" button.
Expected Result:

The selected item should be added to the cart.
Postconditions:

The item is displayed in the cart.
2. Test Case 2: Add Multiple Items to Cart
Objective: To verify that a user can add multiple items to the cart.

Preconditions:

The user is logged in and on the product page.
Test Steps:

Select multiple products.
Click on the "Add to Cart" button for each selected product.
Expected Result:

All selected items should be added to the cart.
Postconditions:

All selected items are displayed in the cart.
